# Heroes Battle - Алгоритмы управления армией

Итоговый проект по курсу "Алгоритмы и структуры данных". Реализация системы управления армией для пошаговой стратегии с использованием алгоритмов оптимального выбора, поиска пути и симуляции боя.

## Реализованные алгоритмы

### 1. **Генерация армии противника** (`GeneratePresetImpl.java`)
**Алгоритм:** Жадный алгоритм с сортировкой по коэффициенту эффективности.
- **Сложность:** O(n log n), где n - количество доступных типов юнитов
- **Обоснование:**
  - Сортировка юнитов: O(n log n)
  - Проход по отсортированному списку: O(n)
  - Итог: O(n log n) + O(n) = O(n log n)

### 2. **Симуляция боя** (`SimulateBattleImpl.java`)
**Алгоритм:** Пошаговая симуляция с сортировкой по атаке.
- **Сложность:** O(k × m × log m), где:
  - k - количество раундов
  - m - количество юнитов в бою
- **Обоснование:**
  - Сортировка юнитов в каждом раунде: O(m log m)
  - Обработка каждого юнита: O(m)
  - Итог для k раундов: O(k × (m log m + m)) = O(k × m × log m)

### 3. **Отбор целей для атаки** (`SuitableForAttackUnitsFinderImpl.java`)
**Алгоритм:** Линейное сканирование с проверкой условий видимости.
- **Сложность:** O(r × c), где:
  - r - количество рядов
  - c - максимальное количество юнитов в ряду
- **Обоснование:**
  - В худшем случае проверяем каждого юнита: O(n)
  - Для каждого юнита проверяем блокировку: O(c)
  - Итог: O(r × c²), но обычно O(n) на практике

### 4. **Поиск кратчайшего пути** (`UnitTargetPathFinderImpl.java`)
**Алгоритм:** BFS (поиск в ширину) на сетке 27×21.
- **Сложность:** O(V + E), где:
  - V - количество вершин (567 клеток)
  - E - количество рёбер (до 8 соседей на клетку)
- **Обоснование:**
  - Каждая клетка посещается не более одного раза
  - Проверка 8 направлений для каждой клетки
  - Итог: O(V × 8) = O(V) = O(567) в худшем случае
